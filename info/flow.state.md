# Финальный план упрощения потока (MVP)

## Цель

Упростить рабочий поток `Qdrant-test.json`, перевести диалог с уровня `topic` на `sub-topic`, не усложняя архитектуру.

---

## Принципы MVP

1. Бот = лендинг-консультант, не “глубокий мед-ассистент”.
2. Минимум state, минимум custom-логики.
3. Ветвление только по структуре файла базы знаний.
4. Реальные баги решаем правилами промпта, а не разрастанием state.
5. Если сомнение — проще и безопаснее: краткий ответ + запись/админ.

---

## Типы файлов базы (главное правило)

### Тип A: без подтем
Признак: в документе только `### Коротко`.

Поведение:
- дать краткий ответ по теме;
- `followup_options: []`;
- мягко предложить запись или перевод на администратора.

### Тип B: с подтемами
Признак: в документе есть `### Коротко` + дополнительные `### ...`.

Поведение:
- сначала ответ по `### Коротко`;
- затем предложить продолжение по нераскрытым подтемам (кнопки или текстом);
- после раскрытия подтем — предложить запись.
