# Flowise / AgentFlow — открытые вопросы и чеклист

Документ для разборки **позже**: что проверить в интерфейсе Flowise, без необходимости помнить весь чат. Отмечайте галочками по мере проверки.

---

## 1. Router (Custom Function после Start / перед IF)

- [ ] Удалён старый флаг **`quiz_active`** (квиз снят с проекта).
- [ ] Добавлена проверка **`situation_pending`**: пока пользователь в сценарии «Рассказать о своей ситуации» и ждётся один ответ — retrieval выключён (результат Router = `"false"`).
- [ ] **`leadIntent !== "none"`** → retrieval `"false"` (воронка записи без RAG).
- [ ] Если используется отдельный **Intent Classifier** и в state есть **`intent`**: для ветки без RAG при `intent` не равном `new_question` — как в вашем графе.
- [ ] Результат Router пишется в **`$flow.state.should_retrieve`** строками **`"true"`** / **`"false"`** (как ожидает IF-нода).

---

## 2. IF-нода (после Router)

- [ ] Условие: **`should_retrieve`** равно строке **`true`** (тип **String**, если Router отдаёт строки).
- [ ] Порядок нод: **Router → IF → …** (IF видит значение, которое только что записал Router).

---

## 3. Qdrant Retrieval

- [ ] В запросе к базе используются **`{{ $flow.state.meta_topic }}`**, **`{{ $flow.state.meta_subtopic }}`**, **`{{ question }}`** — ок по идее.
- [ ] **Запаздывание topic/subtopic:** на момент retrieval в state часто лежат тема и подтема **с прошлого ответа** бота (новые значения попадают в state после **Parse**). Нужно ли что-то менять (упростить запрос, добавить шаг до RAG) — **решение на тестах**.
- [ ] Формат выхода: **Text with Metadata** (как задумано для дальнейшей обработки).

---

## 4. Custom Function сразу после ретривера (фильтр документов)

- [ ] Вход функции (**`$retrieved`** или другое имя) **совпадает** с тем, как Flowise подставляет выход ноды Qdrant в эту функцию. Если имя другое — в коде нужно поправить переменную.
- [ ] Результат функции **сохраняется в state** в поле **`last_context`** (через «Update Flow State» у этой ноды или следующей — как у вас принято). Без этого основной LLM / Parse не увидят контекст базы.
- [ ] **Пустой strict-фильтр:** если включён узкий фильтр по topic+subtopic, но документов 0, сейчас может остаться полный список чанков. Если в ответах появится «мусор» из чужих тем — доработать логику: при пустом strict переходить на фильтр только по **topic** или на другой fallback.

---

## 5. Load Conversion Map (`customFunctionAgentflow_1`)

**Нормальный режим (репозиторий):** после `node scripts/sync-maps.js` берите **целиком** файл [info/load_conversion_map.mvp.js](load_conversion_map.mvp.js) и вставляйте в тело ноды **Javascript Function** в Flowise. Содержимое **автособирается** из `cta_map.json` + `video_map.json` и совпадает по смыслу с **Parse** (модель видит те же `cta_label` / `video` для ключа контекста).

- [ ] Нода подключена в граф **до** Main LLM, в промпте есть `{{customFunctionAgentflow_1}}`.
- [ ] После каждого `sync-maps.js` обновляйте и Parse, и эту ноду (два файла из `info/`).
- [ ] Старую ручную `MAP` с **quiz** — **не использовать** (квиз снят).

**Про `last_context`:** функция читает `$flow.state.last_context`. Если retrieval не было, контекст пустой → чаще отдаётся пакет **`default`** (глобальный CTA, без видео). Для пользователя финал всё равно задаёт Parse по `meta_topic` / `meta_subtopic`.

---

## 6. Main LLM + Parse

- [ ] System prompt = актуальный **`info/prompt.mvp.md`**.
- [ ] Structured output совпадает с полями, которые ждёт **`info/parse_validate_save.mvp.js`**.
- [ ] После правок **`config/cta_map.json`** или **`config/video_map.json`**: выполнить **`node scripts/sync-maps.js`** и **заново вставить весь** скрипт Parse в ноду Flowise.

---

## 7. Start / state (инициализация)

- [ ] Есть **`situation_pending`** (например `false`) — когда сценарий «ситуация» будет подключён в Parse.
- [ ] Нет лишних ключей от старого **квиза** (`quiz_active` и т.п.).

---

## 8. Поле `tone`

- [ ] Отдельный state под **`tone` не обязателен**: тон в финальном JSON ответа, правило привязано к **`meta_topic`** в Parse. Менять ничего не нужно, если так устраивает.
