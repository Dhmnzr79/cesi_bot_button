# Конспект обсуждения: поток бота в Flowise (AgentFlow V2)

Документ фиксирует **что обсуждали** и **к чему сошлись**. Не ТЗ на код — опорные решения и договорённости.

---

## 1. Платформа и ограничения

| Решение | Комментарий |
|--------|-------------|
| **Только AgentFlow V2** | Никакого Chatflow и «древних» нод. Prediction API — с ID именно AgentFlow. |
| **Сейчас фокус внутри Flowise** | Виджет не трогаем; смотрим ответы, JSON, многошаговый диалог по темам. |

---

## 2. Инфраструктура (уже есть)

- **RAG:** Qdrant, метаданные (`topic`, `type`, `subtopic`, `filename` и т.д.). Доп. фильтры позже; старт с **минимального набора**.
- **Заявки:** backend + почта настроены; Flowise заявки не шлёт.
- **Виджет:** отдельный контракт JSON (`widget.js`), доработка позже. Архитектура: `info/bot.md`.

---

## 3. Контракт ответа (legacy JSON)

**Подтверждено:** формат как в старом боте; **Structured / JSON Output в нодах настроить заново**.

Поля: `answer`, `ui_ctaIntent`, `meta_confidence`, `flags_emotional`, `leadIntent`, `meta_shouldHandoff`, `meta_topic`, `followup_options`.

---

## 4. Логика потока (смысл)

1. Вход + память: `chat_history`, `leadIntent`, `last_context`, при необходимости `meta_topic`.
2. **LLM-классификатор:** `intent` = `new_question` | `confirmation` | `rejection` (по смыслу и истории, **не** список слов).
3. **Condition / Router** — нужен (нода или Custom Function).
4. **Qdrant:** только при `new_question` **и** `leadIntent === none`; иначе **обход ретривера**.
5. **`last_context`:** срез знаний после retrieval; при новом содержательном вопросе — **новый поиск** и обновление.
6. **Conversion map** по теме; тест на паре тем (PDF, офферы).
7. **Основной LLM:** `rules.legacy` + `prompt.legacy` + плейсхолдеры.
8. **JSON Output** + при необходимости post-валидация.
9. **Сохранение state** в конце шага — иначе рассинхрон.

**Воронка:** приоритет `leadIntent` над ошибками классификатора на имени/телефоне (как в legacy).

---

## 5. Риски: реально / спорно / натянуто

**Реально:** рассинхрон state; динамический filter Qdrant; широкая ветка без retrieval грузит промпт; «грязный» `last_context` без сжатия.

**Спорно:** авто-сброс `leadIntent` при смене темы — может бить по конверсии; нужны явные правила.

**Натянуто:** «схема минимизирует галлюцинации» — без проверяемых правил и RAG это преувеличение.

**Edge:** первое «Да» при пустом `last_context` — редко, fallback-реплика.

---

## 6. `meta_confidence`

Оставить **скромно:** только тон — категоричность, оговорки, мягкость CTA в тексте.  
**Не** использовать в одиночку для: доверия к ответу, `booking`, безопасности, `handoff` — там правила + RAG + `meta_shouldHandoff` / флаги / воронка.

---

## 7. `flags_emotional`

В одном ответе с `answer` как «усилитель» — слабо. Имеет смысл при **переносе между ходами** или **логах**; без ветвления потока — опционально.

---

## 8. База и конверсия

- Вся база в Qdrant.
- Карта конверсии — пара тестовых тем с PDF/офферами.
